发表于 2014-06-27

在上海学到了很多，也更加了解了信息安全这个专业以及信息安全领域所涉及到的各种各样的“信息”，虽然在此之前对于所学专业的认识已经算是比较全面的了，但是从这次上海之行来看，还是深一层次的了解到了前景发展。

说到信息安全，我经常到看雪，法克等等论坛交流，广大程序猿都吐槽做安全的没有前端搞开发的薪水多，这是为什么？开玩笑说道：“你也不想想搞开发的要加多少班，工作力度多么的大！”只是笑笑，搞安全的怎么了，你们开发或者什么的做的不够好，我们来帮你们完善不足之处，这难道不比你们水平更高么？额，我这里只是随便说的某些方面而已。我了解的安全方面大概主要的也就几个方面：Web安全、软件安全、物联网安全….不过我只了解Web和软件安全，还是对于逆向比较有兴趣，渗透方面总觉得不是那么的那个，就是太杂了，虽然逆向的确很是复杂，我也只是学了16位的汇编语言32位稍微看了看罗云彬写的《win32汇编语言程序设计》，然后就开始了我的逆向道路，不过PC端方面不知道以后会不会逐渐衰弱掉，而移动端非常火热，所以浩哥叫我如果有兴趣可以搞搞安卓安全，用户广，还处于初期阶段，ios显然没有安卓用户多，系统安全方面内核方面笑笑，rootkit还是蛮有兴趣的，额，其实弄加密与解密软件安全与小时候总想着破解使用免费软件有关，现在其实还在想我到底该干嘛，安全是一个很大的领域，你可以理论搞得非常好，去研究信息安全理论标准之类的，另外就是实践技术了，想必大部分人都是属于第二种类型的，这就要求必须得掌握精通的技术才能够找到好的工作。然而学校教学制度不能够造就许多人才，人才又分为研究型人才和技术型人才，比如有些人参加ACM比赛能够获得很好的名次，而让他到公司去写商业程序才发现代码中根本不需要用到高效算法和一些特殊的数据结构，这就是Research和Development的区别，有时候学校组织的某些比赛也差不多是这类性质的。等等，我有太多想说的了…

如果ACM成绩非常好，那么进入这些国际性大公司的概率会非常高，哪怕去不了大公司，中小型公司也非常欢迎这种基础理论扎实的人才，因为他们可以解决别人解决不了的难题，但是这类人才通常要价比较高，很多中小型公司难以承受，同时中小型公司并没有独立的研究部门，所以这类人才进入公司以后通常都是兼有研究和开发两重责任，当然也是处于公司人才金字塔的顶端，享受特殊待遇；开发性人才则是各个类型的公司都需要的，相对来说选择面广一些，但是初始阶段的收入会比研究性人才低一些。你需要根据你的情况来给自己定位，如果是做研究性人才，那么就继续把ACM弄好，多参加ACM大赛，特别是正式的国际性ACM大赛，争取拿奖，同时做好进行研究生学习的的准备（ACM成绩好的话一般都会被科研院所免试录取），在研究生阶段进行进一步的理论研究和培养自己的科研能力，这种方向的话不需要管什么java
web开发、Windows程序开发这类的实现性技术，只需要把算法、模式识别等理论课程学好即可；如果你的定位是开发性人才则把主要精力放到java web开发、Windows程序开发等上边来，如果从事这个方向，因为你已经有一定的ACM的基础了，所以应付面试笔试中的那些算法难题不在话下，那么就在把一部分精力继续投入在ACM的学习上的同时，把更多的精力投入到这些商业程序开发的技术上去。

好吧，以上是个人感慨和对专业和计算机方面的一些感想。下面说说这次上海了解到了什么，其实第一天去了解ARM和FPGA等东西我觉着对我没有太大的启发和影响，因为我压根没有想过在软硬件结合的方面去高安全，当然以后或许做安全会碰到，意思是我不想搞硬件，并不是不喜欢搞那个安全啊，话说的有歧义请谅解，因为嵌入式在我们生活中是无处不在的，不可能脱离它去搞有价值的安全的，话不能说太满，这次去上海个人觉得只是稍微给我们说了些基础的东西并没有深入的讲解，毕竟只有一点时间是在认知的，所以大部分的东西还是要自己去学习去了解。不否认做硬件在某些方面更加的吃香，

就跟Linux和windows区别一样，做Linux总是需求比较大的当然相比windows也相对简单了点。还有就是第二天早上在什么基地的砍伐，其实好多东西我自己都弄过，我就奇怪了，那么多的同学觉得好惊奇，比如那个灰鸽子，其实现在大部分的杀毒软件都能够检测出来，那里只是做个示范，看到安恒信息我又笑了，去年参加过安恒信息赞助的杭州电子科技大学的CTF比赛，排名惨不忍睹，谁让上海交大的0ops队伍和浙江大学还有等等名队参加了呢，一直比较喜欢蓝莲花战队，其实我不知道为什么我们学校就没有组织参加这类比赛的，虽然称不上是什么高端大气上档次的比赛，但至少让感兴趣的同学在计算机的海洋中寻找快乐啊，真的，做哪些题目真的蛮快乐的。

哎。中国自主开发的系统还是模仿的xp系统，或许是为了让国内用户习惯吧，毕竟大家都习惯了windows操作系统了，其实不管做什么方面我认为最重要的还是要有好的编程能力和开发经验语言有很多种，不求多只求精，更何况我这个还不精的呢……汇编语言其实更加考验思维能力，亲眼看过人家用汇编将几百M的视频压缩成了几K大小，这种汇编大赛也是很有意思的，其实如果真的能够在学习中找到了乐趣，也就不会花多时间在游戏上了，而是在专业技术上，倘若一些基础都不懂，估计连逛论坛都不知道别人在说什么了。

浩哥总说我们任务进度太慢了，说我们大一不重视不着急，我其实想说一个星期那么多课，加上晚自习，晚上回去搞搞还断网，时间真的不多，我们也想多花时间在上面，也很着急，大一也就这么过去了，并不像浩哥说的那样我们还小…..我们一眨眼就毕业了，还是要抓紧时间。

WIN7下配置JDK环境变量

发表于 2014-02-28

最近想自学Android，于是下了个JDK，下面是在安装过程中遇到的一些问题，防止以后忘记，所以整理了一下。

1.下载JDK

下载地址：http://www.oracle.com/technetwork/java/javase/downloads/jdk7-downloads-1880260.html
在这个下载页面可以看到windows有两个版本的JDK：
Windows x8689.09 MB
Windows x6490.66 MB
x86是32位操作系统，x64是64位的操作系统。
我的电脑是64位win7系统，所以我下载的版本是 jdk-7u51-windows-x64.exe下载完成后，安装 。
在网上看到了JRE和JDK的区别，虽然不是太理解但是想先记在这里：
JRE为Java Runtime Environment的简称，JavaRuntimeEnvironment(JRE)是可以在其上运行、测试和传输应用程序的Java平台。它包括Java虚拟机、Java核心类库和支持文件。它不包含开发工具—编译器、调试器和其它工具。JRE需要辅助软件—Java Plug-in—以便在浏览器中运行applet。
JDK（Java Development Kit）是Sun Microsystems针对Java开发员的产品。自从Java推出以来，JDK已经成为使用最广泛的Java SDK。JDK 是整个Java的核心，包括了Java运行环境、Java工具和Java基础类库。

2.环境变量的配置

1.右击“我的电脑”，点击“属性”：选择“高级系统设置”选项卡，点击“环境变量”： 　　
2.在“系统变量”中，设置3项系统变量，JAVA_HOME,PATH,CLASSPATH （注：“系统变量”对所有用户有用，而“用户变量”则只对当前登陆用户有用。对于变量的大小写，是不进行区分的。）
3.因为是默认安装，所以JDK1.7是安装在C:\Program Files\Java\jdk1.7.0_25里了。现在为三个变量设值；

新建 JAVA_HOME 设值 C:\Program Files\Java\jdk1.7.0_25

新建 CLASSPATH 设值 .;%JAVA_HOME%/lib;%JAVA_HOME%/lib/tools.jar

编辑 PATH 设值 ;%JAVA_HOME%/bin;%JAVA_HOME%/jre/bin

注意事项：

1.配置JAVA_HOME变量的作用：JAVA_HOME路径下包括lib，bin，jre等文件夹，以后配tomcat，eclipse等java开发软件可能需要依赖此变量。

  2.在设置系统变量Path（系统已经有），要注意前面的分号“;”,配置Path的作用：Path使系统可以在任何路径下识别java命令。

  3.在设置系统变量CLASSPATH时，要注意CLASSPATH中有一英文句号“.”后跟一个分号，表示当前路径的意思,配置CLASSPATH变量的作用： 为系统指明java加载类(class or lib)的路径，只有类在classpath中，java命令才能识别，其中，%JAVA_HOME%就是引用前面指定的JAVA_HOME。

  4. 用户变量的设置必须注销重登录后才起作用，但是当本次配置中所有环境变量都是系统变量，而设置系统变量的则不用。

C：/> java
程序是不能直接运行的，因此，我们得对它先进地编译，即输入javac Hello.java如下：

这样就编译好了，如果没有编译好的话，它会提示错误。 java文件被编译好了之后，就会自动生成一个CLASS编译文件，因此我们可以运行它，只要输入“java Hello”就可以了。
